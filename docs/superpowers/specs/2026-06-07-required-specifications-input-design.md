# 规格参数随调用输入设计

## 决策

用户在调用 Skill 时随产品图片一起提供尺寸和规格文字。Skill 不主动询问，也不支持第七张待补充或先交付六张的状态。

默认一次生成并交付完整的 7 张 `1:1` 商品图，其中：

`07-specifications-square.png` 使用用户输入的真实参数生成经典参数表、尺寸线和测量说明。

## 输入与生成

- 接受灵活的 `字段：内容` 格式。
- 至少应包含一项尺寸字段，例如长、宽、高、直径、口径、底径或厚度。
- 数值、单位和“约”等限定词按用户原文使用。
- 不从商品照片估算尺寸、容量、重量、材质纯度或工艺。
- 默认免责声明为：`尺寸为手工测量，存在合理误差，请以实物为准`。

## 严格交付

- 删除 `--specifications-awaiting-input` 参数。
- 删除 `availability: "awaiting-user-input"` 状态。
- manifest 仍使用 schema v2，并默认包含 7 条资产记录。
- 所有 7 张图都必须为 `complete` 且文件存在。
- 任意图片为 `pending`、`failed`、缺失或比例不符时，整套验证失败。

## 文档

`SKILL.md` 和 `README.md` 明确告诉使用者：调用时应直接附上规格尺寸文字，Skill 随后一次性交付完整七图。
